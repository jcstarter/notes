Git Notes

  • Git is a version control system that runs on your computer (local) to track changes
  • Projects, aka, Repos can be pushed to a server - services like GitHub and BitBucket
  • Git has three main states that files can reside in: committed, modified, and staged
  • To start tracking changes locally, use: cd /Users/JC/directory/folder
    Then run: git init
  • Add the files you want git to track with: git add FILE for a single file
    or: git add --all or: git add -A for all the files in the folder
  • After git add, files are now staged. To commit, use: git commit -m 'Commit message'
  • If commit message is forgotten, save and exit by pressing: Esc and then :x
  • Run: git status to check status after action
  • Branches: git branch branch-name (This command creates a branch but does not switch you to that branch)
  • To checkout branch just created: git checkout branch-name
  • To straight up checkout a branch: git checkout -b branch-name
  • Simple merging - Start by checking out master branch: git checkout master
  • Then, merge in branch-name: git merge branch-name
  • If you donâ€™t plan on using that branch anymore, you can delete: git branch -d branch-name

Git Related Notes

  • It is recommended to create a readme file, either as: .txt or: .md (markdown language)
  • To push the project to GitHub: git remote add origin remote repository URL then: git push -u origin master
  • To pull down from a repo: git pull --all
